Internet Project Minimum Requirements

Use this checklist to ensure your project meets the required technical and design criteria.

This page focuses on the minimum expectations. The goal is clarity, not feature count.

General Guidelines

  • Technologies Used: HTML5, CSS3, JavaScript, C#, .NET Minimal API
  • Use javascript, html, css for the frontend and c# for the server code. .NET Minimal API (like this web) is fine.
  • The web app should be well planned and easy to understand.
  • Include multiple pages and make sure one of them is an About page.
  • All pages should share the same header, navigation, and footer.
  • Submission deadline: 24/07/2026.

HTML Requirements

Your project must demonstrate meaningful use of at least the following HTML elements:

  • Headings - <h1>, <h2>, <h3>
  • Images - <img>
  • Links - <a>
  • Tables - <table>
  • Lists - <ul>, <ol>, <li>

This site uses some more and you are invited to explore and be inspired

CSS Requirements

  • Use one main style.css file.
  • Do not place CSS directly in HTML elements.
  • Pages should share a common theme
  • Show some layout work such as display, position, or responsive design.

JavaScript Requirements

  • Place the main JavaScript in script.js.
  • Use JavaScript on all pages.
  • Include some dynamic features. This site uses js for displaying date, information when hovering over certain elements and loading data from the server.

Server-Side Requirements

  • Server-side code must be written in C#.
  • Use .NET Minimal API, Web API, Razor Pages, or Blazor.
  • Send and receive data between the browser and the server.
  • Using .NET Minimal API, this sample uses a few simple POST/GET endpoints which you can view in program.cs .
