About This Project

This site is intentionally small. It is intended to serve as a teaching by example thing that you can use to accelerate understanding.

Project Purpose

The project serves two purposes:

  • Explain what you are expected to build.
  • Show a basic example of a multi-page web with a C# backend.

How It Is Built

Part What it does
HTML pages Provide the content for Home, About, Requirements, and Example.
CSS file Keeps the layout clean and consistent without a complex design.
JavaScript file Handles a few basic tasks such as the date and one API call.
C# file Serves the pages and returns some data for the browser to display.

HTML of the elements Used

Element Usage
<h1>, <h2>, <h3> Page titles and section headings
<p> Body text and descriptions
<a> Navigation links and internal references
<img> Illustrative image on the example page
<table> Requirement summaries and page structure
<ul>, <ol>, <li> Requirement lists and steps
<header>, <nav>, <footer> Shared layout across pages

Why It Is Simple

The code avoids extra features so you can focus on the basics: page structure, styling, simple JavaScript, and a bit of c#. The learning curve of c# .NET Minimal API is gentle. You write C# code in a single file, define routes with app.MapGet and app.MapPost, and the server handles the rest.

What JavaScript Does

  • Shows the current date.
  • Highlights the active navigation link.
  • Fetches the greeting, server time, and current message from the server.
  • Show some text when mouse hovers certain elements

Server-Side Implementation

The C# backend uses .NET Minimal API. It exposes three GET routes and one POST route:

  • GET /api/greeting returns a welcome message stored in a C# variable.
  • GET /api/current-time returns the current server date and time.
  • GET /api/message returns the current message stored on the server.
  • POST /api/message updates the stored message sent from the browser form.

Built with HTML, CSS, JavaScript, and C# .NET Minimal API.
